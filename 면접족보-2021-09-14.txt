1. 자바의 명명 규칙에 대하여 설명하시오.

-클래스
1) 반드시 대문자로 시작
2) 생성자 이름은 클래스 이름과 같이 반드시 대문자로 시작
3) 클래스 이름을 만들 때는 해당클래스가 어떤 목적으로 만들어졌는지 유추 가능하도록 만든다.

-메소드
1) 소문자로 시작
2) 여러 단어로 조합되면 첫 단어를 제외하고 다음 단어의 첫글자는 대문자로 시작
3) 보통 메소드는 클래스 기능을 담당하므로 동사로 시작

-변수
1) 첫 번째 문자가 소문자인 명사로 정함
2) 뜻이 있는 단어를 사용한다, 이 변수가 무엇인지 알아볼 수 있도록.
3) 여러 단어로 구성된 이름의 경우 두 번째 단어의 첫 글자는 대문자로 표기함
4) _는 사용하지 않는다.

2. 아래의 접근 제한자에 대하여 설명하시오.

-private  : 현재 객체, 클래스 내에 있으면 허용 -------- private를 꼭 쓰는 이유는 객체지향 언어이기 때문에 객체의 특성을 살리기 위해서 씀

-protected : 동일 패키지(폴더)에 있는 객체와 상속관계의 객체들만 허용 

-default  : 동일 패키지(폴더)에 있는 객체들만 허용

-public : 모든 접근을 허용, 변수도 마찬가지 -------- 한개의 파일안에는 외부에서 접근가능한 클래스를 여러주기 위해 1개의 public 클래스만 선언함.

3. 지역 변수에 접근 제한자를 붙이지 않는 이유는?

접근 제한자는 클래스, 생성자, 멤버변수, 멤버메소드에만 쓰인다.

4. 캡슐화(encapsulation)에 대하여 설명하시오.

클래스의 내부에 선언된 데이터를 보호하기 위해서이다.
데이터가 유효한 값을 유지하도록 또는 비밀번호와 같은 데이터를 외부에서 함부로 변경하지 못하도록 하기 위해서는 외부로부터의 접근을 제한하는 것이 필요하다
이것이 캡슐화다
